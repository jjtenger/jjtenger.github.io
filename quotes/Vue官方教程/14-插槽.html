  • 14-插槽
  • 插槽内容
    使用 <slot></slot> 在组件模版中定义插槽。当组件渲染的时候，插槽内容将会被替换为组件起始标签和结束标签之间的内容。
    插槽无非就是在子组件中挖个坑，坑里面放什么东西由父组件决定（渲染时决定）。
    插槽内可以包含任何模板代码，包括 HTML，甚至其它的组件。
    如果父组件没有包含一个 <slot> 元素，则该组件起始标签和结束标签之间的任何内容都会被抛弃。
  • 编译作用域
    父级模版向子级模版插槽中填充内容时，只能访问父级模版的实例属性（即作用域），但不能访问子集模版的作用域。
    父级模板里的所有内容都是在父级作用域中编译的；子模板里的所有内容都是在子作用域中编译的。
  • 后备内容
    在 <slot></slot> 标签中设置插槽的后备内容。当父级组件向子级标签填充但不提供任何插槽内容时，后备内容将会被渲染。
  • 具名插槽
    使用 <slot> 元素的 name 特性定义具名插槽。
    向具名插槽填充内容时，使用 v-slot 指令，并以 v-slot 的参数的形式（:name）指定 <template> 元素将被填充到哪个插槽。
    任何没有被包裹在带有 v-slot 的 <template> 中的内容都会被视为默认插槽的内容。
    仍然可以用 v-slot:default 在一个 <template> 中包裹默认插槽的内容。
    注意 v-slot 只能添加在一个 <template> 上 (只有一种例外情况 )，这一点和已经废弃的 slot 特性不同。
  • 作用域插槽
    为了让插槽内容（父级组件提供的内容）能够访问子组件中的数据，可以在 <slot> 元素的特性上绑定子组件对象。
    绑定在 <slot> 元素上的特性被称为 插槽 prop 。
    在填充内容时，父级组件通过指定 v-slot 的值拿到包含所有插槽 prop 的对象，该值可以任意指定。
    • 独占默认插槽的缩写语法
      当提供的内容只有默认插槽时，组件的标签才可以被当作插槽的模板来使用。这样我们就可以把 v-slot 直接用在组件上。
      就像假定未指明的内容对应默认插槽一样，不带参数的 v-slot 被假定对应默认插槽。
      注意，默认插槽的缩写语法不能和具名插槽混用，因为它会导致作用域不明确。
    • 解构插槽 Prop
      作用域插槽的内部工作原理是将你的插槽内容包括在一个传入单个参数的函数里（该参数即包含所有插槽 prop 的对象）。
      这意味着 v-slot 的值实际上可以是任何能够作为函数定义中的参数的 JavaScript 表达式。
      所以在支持的环境下（单文件组件或现代浏览器），你也可以使用 ES2015 解构来传入具体的插槽 prop。（通过表达式解构传入的函数参数）
  • 动态插槽名
    动态指令参数也可以用在 v-slot 上，来定义动态的插槽名。
  • 具名插槽的缩写
    跟 v-on 和 v-bind 一样，v-slot 也有缩写，即把参数之前的所有内容 ( v-slot: ) 替换为字符 # 。
    然而，和其它指令一样，该缩写只在其有参数（明确的插槽名）的时候才可用。
  • 其它示例
    插槽 prop 允许我们将插槽转换为可复用的模板，这些模板可以基于输入的 prop 渲染出不同的内容。 这在设计封装数据逻辑同时允许父级组件自定义部分布局的可复用组件时是最有用的。
