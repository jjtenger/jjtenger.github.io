  • 11-组件注册
  • 组件名
    在注册一个组件的时候，我们始终需要给它一个名字。
    全局注册中，该组件名就是 Vue.component 的第一个参数。
    • 组件名大小写
      强烈推使用 kebab-case，荐遵循 W3C 规范 中的自定义组件名 (字母全小写且必须包含一个连字符)。
      • 使用 kebab-case
      • 使用 PascalCase
  • 全局注册
    使用 Vue.component 来创建组件。
  • 局部注册
    通过一个普通的 JavaScript 对象来定义组件。
    然后在 components 选项中定义你想要使用的组件。
    对于 components 对象中的每个属性来说，其属性名就是自定义元素的名字，其属性值就是这个组件的选项对象。
    注意，局部注册的组件在其子组件中不可用。
  • 模块系统( Babel 和 webpack )
    • 在模块系统中局部注册
      推荐创建一个 components 目录，并将每个组件放置在其各自的文件中。
      首先 import / require 导入每个你想使用的组件，然后 export default components。
    • 基础组件的自动化全局注册
      如果你使用了 webpack (或在内部使用了 webpack 的 Vue CLI 3+ )，那么就可以使用 require.context 只全局注册这些非常通用的基础组件。
      在应用入口文件 (比如 src/main.js) 中全局导入基础组件的示例代码。
      记住，全局注册的行为必须在根 Vue 实例 (通过 new Vue ) 创建之前发生。
