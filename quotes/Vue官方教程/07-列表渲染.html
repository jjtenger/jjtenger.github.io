  • 07-列表渲染
  • 用 v-for 把一个数组对应为一组元素
    使用 item in items 形式的特殊语法。
    v-for 块中，我们可以访问所有父作用域的属性。
    v-for 还支持一个可选的第二个参数，即当前项的索引。
    也可以用 of 替代 in 作为分隔符。
  • 在 v-for 里使用对象
    可以用 v-for 来遍历一个对象的属性。
    可以提供第二个参数作为对象的属性名，第三个参数作为索引。
  • 维护状态
    当 Vue 正在更新使用 v-for 渲染的元素列表时，默认使用“就地更新”的策略。
    只适用于不依赖子组件状态或临时 DOM 状态 (例如：表单输入值) 的列表渲染输出 。
    为了给 Vue 一个提示，以便它能跟踪每个节点的身份，从而重用和重新排序现有元素，你需要为每项提供一个唯一 key 属性。
    不要使用对象或数组之类的非基本类型值作为 v-for 的 key。请用字符串或数值类型的值。
  • 数组更新检测
    • 变异方法 (mutation method)
      Vue 将被侦听的数组的变异方法进行了包裹，所以它们也将会触发视图更新。
    • 替换数组
      非变异方法总是返回一个新数组，使用此类方法时，可以用新数组替换旧数组。
    • 注意事项
      Vue 不能检测以下数组变动：items[indexOfItem] 设置数组项、 items.length 修改数组长度。
      可以使用 Vue.set 和 Array.prototype.splice 方法代替，实现响应式更新。
  • 对象变更检测注意事项
    还是由于 JavaScript 的限制， Vue 不能检测对象属性的添加或删除。
    可以使用 Vue.set(object, propertyName, value) 方法向嵌套对象添加响应式属性。
    使用 Object.assign() 添加多个响应式属性。
  • 显示过滤/排序后的结果
    创建一个计算属性或者方法来返回过滤或排序后的数组。
    计算属性 vs 侦听属性
  • 在 v-for 里使用值范围
    v-for 也可以接受整数。在这种情况下，它会把模板重复对应次数。
  • 在 template 元素上使用 v-for
    类似于 v-if ，你也可以利用带有 v-for 的 <template> 来循环渲染一段包含多个元素的内容。
  • v-for 与 v-if 一同使用
    注意，我们不推荐在同一元素上使用 v-if 和 v-for 。
    当它们处于同一节点，v-for 的优先级比 v-if 更高，这意味着 v-if 将分别重复运行于每个 v-for 循环中。
  • 在组件上使用 v-for
    在自定义组件上，你可以像在任何普通元素上一样使用 v-for 。
    然而，任何数据都不会被自动传递到组件里，因为组件有自己独立的作用域。为了把迭代数据传递到组件里，我们要使用 prop。
    此外，子组件可以使用 $emit 触发父组件的自定义事件。
