列表结构

Python语言及其应用-第3章列表、元组、字典与集合（P1-2）

列表结构——由任意类型元素组成的序列。
除字符串外， Python 还有另外两种序列结构：元组和列表。它们都可以包含零个或多个元素。与字符串不同的是， 元组和列表并不要求所含元素的种类相同，每个元素都可以是任何 Python 类型的对象。得益于此，你可以根据自己的需求和喜好创建具有任意深度及复杂度的数据结构。
为什么 Python 需要同时设定列表和元组这两种序列呢？这是因为元组是不可变的，当你给元组赋值时， 这些值便被固定在了元组里，再也无法修改。然而，列表却是可变的，这意味着可以随意地插入或删除其中的元素。
列表非常适合利用顺序和位置定位某一元素，尤其是当元素的顺序或内容经常发生改变时。你可以直接对原始列表进行修改：添加新元素、删除或覆盖已有元素。在列表中，具有相同值的元素允许出现多次。

使用[offset]获取或者修改元素

Python语言及其应用-第3章列表、元组、字典与集合（P3-5）

和字符串一样，通过偏移量可以从列表中提取对应位置的元素。同样，负偏移量代表从尾部开始计数。

指定的偏移量对于待访问列表必须有效——该位置的元素在访问前已正确赋值。当指定的偏移量小于起始位置或者大于末尾位置时， 会产生异常（错误）。
就像可以通过偏移量访问某元素一样，你也可以通过赋值对它进行修改。同样，列表的偏移量必须是合法有效的。
通过这种方式无法修改字符串中的指定字符， 因为字符串是不可变的。列表是可变的。

使用insert()在指定位置插入元素

Python语言及其应用-第3章列表、元组、字典与集合（P6）

append() 函数只能将新元素插入到列表尾部，而使用 insert() 可以将元素插入到列表的任意位置。指定偏移量为 0 可以插入列表头部。如果指定的偏移量超过了尾部，则会插入到列表最后，就如同 append() 一样，这一操作不会产生 Python 异常。

使用del删除指定位置的元素

Python语言及其应用-第3章列表、元组、字典与集合（P6-7）

当列表中一个元素被删除后，位于它后面的元素会自动往前移动填补空出的位置，且列表长度减 1。
del 是 Python 语句，而不是列表方法——无法通过 marxes[-2].del() 进行调用。 del 就像是赋值语句（=）的逆过程：它将一个 Python 对象与它的名字分离。如果这个对象无其他名称引用，则其占用空间也被会清除。

使用sort()重新排列元素

Python语言及其应用-第3章列表、元组、字典与集合（P9）

在实际应用中，经常需要将列表中的元素按值排序，而不是按照偏移量排序。 Python 为此提供了两个函数：

  • 列表方法 sort() 会对原列表进行排序，改变原列表内容；
  • 通用函数 sorted() 则会返回排好序的列表副本，原列表内容不变。

如果列表中的元素都是数字， 它们会默认地被排列成从小到大的升序。如果元素都是字符串，则会按照字母表顺序排列。
当列表中的所有元素都是同一种类型时（例如 marxes 中都是字符串）， sort() 会正常工作。有些时候甚至多种类型也可——例如整型和浮点型——只要它们之间能够自动地互相转换。
默认的排序是升序的，通过添加参数 reverse=True 可以改变为降序排列
