代理模式

代理模式的作用是：为其他对象提供一种针对真实对象的代理访问机制。

在某些情况下，一个客户不想或者不能直接访问另一个对象，而代理对象可以在客户端和目标对象之间起到中介的作用。

代理模式一般涉及到的角色有：

  • 抽象角色：声明真实对象和代理对象的共同接口
  • 代理角色：代理对象角色内部含有对真实对象的引用，从而可以操作真实对象。其中代理对象提供与真实对象相同的接口以便在任何时刻都能代替真实对象。同时，代理对象可以在执行真实对象操作时，附加其他的操作，相当于对真实对象进行封装。
  • 真实角色：代理角色所代表的真实对象，是我们最终要执行的对象。

如果要按照传统方法使用代理模式，那么真实角色必须是事先已经存在的，并将其作为代理对象的内部属性。但是实际使用时，如果一个真实角色必须对应一个代理角色，大量使用会导致类的急剧膨胀；此外，如果事先并不指定真实角色，该如何使用代理呢？这个问题可以通过Java的动态代理类来解决。

动态代理的概念

动态代理是指客户通过运行期动态生成的代理类来调用目标接口的方法。

Java动态代理类位于java.lang.reflect包下，一般主要涉及到以下两个接口/类：

  • Proxy：动态代理工厂类
  • InvocationHandler接口：代理执行类（代理模式里头的代理角色）

Proxy：动态代理工厂类

其中主要包含以下方法：

protected Proxy(InvocationHandler h)：构造函数，用于给内部执行对象h赋值。

static Class getProxyClass(ClassLoader loader,Class[] interfaces)：返回一个代理类，其中loader是类装载器，interfaces是真实类所拥有的全部接口的数组。

static Object newProxyInstance(ClassLoader loader, Class[] interfaces, InvocationHandler h)：返回代理类的一个实例，返回后的代理类可以当作被代理类使用，可使用被代理类的在interfaces接口中声明过的所有方法，但实际执行的方法体由InvocationHandler h决定。

所谓Dynamic Proxy是这样一种类：它是在运行时生成的类（实际代码中不存在），在生成时提供一组interface给它就会返回一个实现了这些interface的动态类。可以把该类的实例当作这些interface中的任何一个来使用。当然，这个Dynamic Proxy其实就是一个Proxy，它不会替你作实质性的工作，在生成它的实例时你必须提供一个handler，由它接管实际的工作。

一般该接口实现类含有对真实对象的引用，从而可以在执行invoke()方法时调用真实对象的被代理方法。

通过这种方式，被代理的对象(RealSubject)可以在运行时动态改变，需要被代理的接口(Class[] interfaces)可以在运行时改变，控制的方式(InvocationHandler)也可以动态改变，从而实现了非常灵活的动态代理关系。
