事件的概念

GUI是由事件(event)驱动的

当用户与GUI交互可以产生事件(events)，事件是描述发生了什么的对象。存在各种不同类型的事件类用来描述各种类型的用户交互。一些常见的交互方式:移动鼠标、用鼠标点击按钮、在文本框中输入数据、关闭窗口等。

主要涉及三种对象

  • 事件源(An event source)：可供用户进行交互的GUI组件，是一个事件的产生者。例如，在Button组件上点击鼠标会产生以这个Button 为源的一个ActionEvent（事件）。

  • 事件对象(An event object)

  • 封装了包含所发生的各种事件的有效信息。

  • 信息包括: 事件源的引用、以及事件监听器在处理事件时所需要的其它各种信息。

  • 信息被包含在类AWTEvent或其子类的实例对象中。

  • 事件监听器(event listener(s))：等待事件产生，并接收事件、解释事件并处理用户交互的对象。

事件监听器：实现了Listener接口的类

对于每类事件，都有一个监听器接口，这个接口由想接收这类事件的类实现（Listener）。这个接口定义了一个或多个方法。

每种事件监听器接口都是以类似于XxxListener的形式来命名的，如：ActionListener、MouseListener、ItemListener等。

事件监听器的对象会被addListener()方法注册到事件源上。当发生特定的事件时，就会调用监听者的对应方法（事件处理器）。

多监听者可以使一个程序的不相关部分执行同样的动作。事件发生时，所有被注册的监听者的事件处理器都会被调用。

事件Adapter（适配器）

Java中为那些具有多个方法的监听者接口提供了事件适配器类，这个类通常命名为XxxAdapter，在该类中以空方法体实现了相应接口的所有方法。

可通过继承适配器类来编写监听者类，在类中只需给出关心的方法，从而减轻工作量。
