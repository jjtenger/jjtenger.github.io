  • 08-多态POLYMORPHISM

所谓多态就是指程序中定义的引用变量所指向的具体类型和通过该引用变量发出的方法调用在编程时并不确定，而是在程序运行期间才确定，即一个引用变量倒底会指向哪个类的实例对象，该引用变量发出的方法调用到底是哪个类中实现的方法，必须在由程序运行期间才能决定。因为在程序运行时才确定具体的类，这样，不用修改源程序代码，就可以让引用变量绑定到各种不同的类实现上，从而导致该引用调用的具体方法随之改变，即不修改程序代码就可以改变程序运行时所绑定的具体代码，让程序可以选择多个运行状态，这就是多态性。

父类型的引用可以指向子类型的对象。子类是父类，父类可以代指子类。

  • 父类型的引用可以通过指向不同子类对象，然后调用子类对象的对应重写方法来获得父类方法的各种不同的实现，而不用关注子类方法的具体实现途径。（动态绑定方法）；
  • 多态是父类从子类获取属性和方法，而继承是子类从父类获取属性和方法；
  • 多态是通过继承和方法重写来实现的（不是方法重载），是晚绑定（编译期无须确定指向，运行期确定）,而方法重载是编译期的行为;
  • 父类引用不能调用子类中父类不存在的属性与方法。当使用多态方式调用方法时，首先检查父类中是否有对应方法，如果没有则编译错误；如果有，再去调用子类对应的重写方法。

1.多态的两种类型转换

父类型引用可以指向子类型对象，也可以强制转换为子类型引用，但子类型引用不能指向父类型对象。

  • 向上类型转换（upcast）：比如说Cat类型转换为Animal类型。子类型引用可以直接赋值给父类型。对于向上类型转换，不需要显示指定；
  • 向下类型转换（downcast）：比如Animal类型转换为Cat类型。指向子类型对象的父类型引用可以强制转换为指向该对象的子类型引用。但父类型的引用不可以转换为子类型引用（人不可以强制转换为男人）。向下类型转换必须要显式指定（必须使用强制类型转换）。向下类型转换可以使父类型引用调用子类中父类不存在的属性与方法。

在类型转换中，子类可以自动转型为父类，但是父类强制转换为子类时只有当引用类型真正的身份为子类时才会强制转换成功，否则会失败，抛出ClassCastException异常。

2.抽象类与抽象方法（abstract）

抽象类（abstract class）：使用了abstract关键字所修饰的类叫做抽象类。抽象类无法实例化，也就是说，不能new出来一个抽象类的对象（实例）。

抽象方法（abstract method）：使用abstract 关键字所修饰的方法叫做抽象方法。抽象方法有声明无实现（无大括号，注意分号）。抽象方法需要定义在抽象类中。相对于抽象方法，之前所定义的方法叫做具体方法（有声明，有实现）。

抽象类注意事项

如果一个类中包含了抽象方法，那么这个类一定要声明成 abstract class，也就是说，该类一定是抽象类；反之，如果某个类是抽象类，那么该类既可以包含抽象方法，也可以包含具体方法（有声明、有实现）。

无论何种情况，只要一个类是抽象类，那么这个类就无法实例化。

在子类继承父类（父类是个抽象类）的情况下，那么该子类必须要实现父类中所定义的所有抽象方法；否则，该子类需要声明成一个 abstract class。但如果一个子类实现（重写）了父类抽象类的所有抽象方法。它也可以定义为抽象类，只不过无法实例化而已。

abstract 不能与private、static、final或native并列修饰同一个方法。

抽象类的作用

父类抽象类只是定义并规范功能的实现方法及其返回类型，并不涉及方法的具体实现过程。方法的具体实现由实现了其抽象类的子类来完成。通过多态，父类引用可以调用子类的具体实现方法。这里，父类也可以不定义成一个抽象类，其抽象方法也可以用一个无意义的方法来代替，但这样做不规范，不建议这样做。

3.接口（interface）

接口的地位等同于class，可以看作是特殊的抽象类（抽象类中可以有具体方法，也可以有抽象方法，而接口中只能有抽象方法，不能有具体方法）。

接口中的所有方法都是抽象方法，都是public的。在声明接口中的方法时，可以使用 abstract 关键字，也可以不使用。通常情况下，都会省略掉 abstract 关键字。

接口中也可以定义成员变量（）较少使用。其成员变量都是public、final、static的（修饰符同样可以省略）；

类中使用关键字implements实现某个接口；同抽象类，实现接口的类要么是抽象类，要么必须实现接口中所有方法。

Java 是单继承的，也就是说某个类只能有唯一一个父类；但一个类可以实现多个接口，多个接口之间使用逗号分隔。

接口类型的多态与类的多态类似，接口类型的引用可以指向实现该接口的类的实例。关于接口与实现接口的类之间的强制类型转换方式也与父类和子类之间的强制类型转换方式完全一样。
