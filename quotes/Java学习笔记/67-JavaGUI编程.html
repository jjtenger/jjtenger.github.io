GUI 全称是Graphical User Interface，即图形用户界面。根据作用GUI组件可分为基本组件和容器。

  • 组件：又称构件，诸如按钮、文本框之类的图形界面元素。

  • 容器其实是一种比较特殊的组件，可以容纳其它组件如窗口、对话框等。所有的容器类都是java.awt.Container的直接或间接子类。

GUI的设计步骤：

  • 先设计一个窗口，如JFrame。
  • 确定布局管理器。
  • 在窗口中添加所需组件。
  • 增加事件处理。

AWT组件

AWT(Abstract Window Toolkit)，抽象窗口工具包，SUN公司提供的用于图形界面编程(GUI)的类库。基本的AWT库处理用户界面元素的方法是把这些元素的创建和行为委托给每个目标平台上（Windows、Unix、Macintosh等）的本地GUI工具进行处理。例如：如果我们使用AWT在一个Java窗口中放置一个按钮，那么实际上使用的是一个具有本地外观和感觉的按钮。这样，从理论上来说，我们所编写的图形界面程序能运行在任何平台上，做到了图形界面程序的跨平台运行。

AWT特性如下：

  • 提供基本的GUI组件，用在所有的Java applets及应用程序中。
  • 具有可以扩展的超类，它们的属性是继承的。
  • 确保显示在屏幕上的每个GUI组件都是抽象类组件的子类。
  • 所有类在java.awt包下。

AWT组件继承关系

AWT容器（Container）及其子类

Container是Component（组件）的一个子类，两个主要实现类是Window和Panel。

注：容器不但能容纳组件，还能容纳其它容器，这一事实对于建立复杂的布局是关键的，也是基本的。

Java.awt.Window类

Window是显示屏上独立的本机窗口，它独立于其它容器。Window有两种形式：Frame（框架）和Dialog（对话框）。Frame和Dialog是Window的子类。

  • Frame

Frame（框架）是一个带有标题和缩放角的窗口。可以用add()方式来给框架添加组件。框架的缺省布局管理器是Border Layout。它可以用setLayout()方式来改变。框架类中的构造方法Frame(String)用由String规定的标题来创建一个新的不可见的框架对象。当它还处于不可见状态时，将所有组件添加到框架中。在框架显示在屏幕上之前，必须设置成可见的（通过调用程序setVisible(true)），而且其大小是确定的（通过调用程序setSize()或pack()）。

Frame的继承层次：java.lang.Object–java.awt.Component–java.awt.Container–java.awt.Window–java.awt.Frame

  • Dialog

Dialog（对话框）没有菜单条。尽管它能移动，但它不能缩放。

Java.awt.Panel类

Panel包含在另一个容器中，或是在Web浏览器的窗口中。Panel确定一个四边形，其它组件可以放入其中。Panel必须放在Window之中（或Window的子类中）以便能显示出来。

象Frames一样，Panels提供空间来容纳任何GUI组件，包括其它面板。每个面板都可以有它自己的布管理程序。一旦一个面板对象被创建，为了能看得见，它必须添加到窗口或框架对象上，用Container类中的add()方式可以做到这一点。当把Panel作为一个组件添加到某个容器中后，该Panel仍然可以有自己的布局管理器。因此，可以利用Panel使得BorderLayout中某个区域显示多个组件。

Panel类的继承层次：java.lang.Object–java.awt.Component–java.awt.Container–java.awt.Panel

Swing组件

Swing大部分组件是由纯Java Code 所写成的。因此解决了Java 因窗口类而无法跨平台的问题，使窗口功能也具有跨平台与延展性。而且Swing不需占有太多系统资源，因此我们称Swing为lightweight component。

相对于Swing，AWT是hevayweight component。由于AWT具有native Code的C语言成分，所以若想自改动AWT的窗口变化时，必须编写自己的C语言native Code，然后搭配AWT的native Code与JDK函数库存。遇到不同的平台时，又必须重新理发与编译自己所写的native code。因此AWT不具有跨平台特性，耗时且难以理解又耗系统资源，所以就称之为hevayweight component，表示AWT组件是不容易改动的。

在javax.swing包中，定义了两种类型的组件：容器和组件。其中各种组件必须放在容器，容器本身也是一种组件。
