1.包（package）

用于将完成不同功能的类分门别类，放在不同的目录（包）下。其概念类似文件夹。

使用package关键字定义一个包名。语法：

一般将公司域名反转作为包名。包名全部小写，用"."分割。

一个包名对应文件目录下面的一个文件夹，子目录对应的包为子包。

package语句只能是源代码除注释外的第一条语句。

定义类时没有使用package，则认为位于默认包内（default package）。

一个类如果想被其他包中的类访问到，需要加上public关键字修饰，加上了public修饰的类必须和文件名同名。

利用通配符*可以导入包下所有类，但不包括子包下的类。注意，默认包中的类无法使用通配符导入其他包下的所有类。

同一个包下的类不需要导入，直接用类名使用即可；

首先需要定义包（package）（可选），接下来使用 import进行导入（可选），然后才是class或 interface 的定义。

java文件在编译时如果import指定路径下检测不到指定包会编译出错。

2.访问修饰符（access modifier）

  • public（公共的）：被 public所修饰的属性和方法可以被所有类访问。
  • protected（受保护的）：被 protected所修饰的属性和方法可以在类内部、相同包所访问。 并且可以被其它包里的子类所继承，但不能直接调用。
  • private（私有的）：被 private 所修饰的属性和方法只能在该类内部使用。
  • 默认的（不加任何访问修饰符）：在类内部以及相同包下面的类所使用。
