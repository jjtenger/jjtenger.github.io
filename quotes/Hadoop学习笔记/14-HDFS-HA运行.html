启动JournalNode

第一次格式化HDFS的过程中，HA需要journalnode通讯，所以需要先把三个节点的journalnode启动。

在node1节点上执行hadoop-daemons.sh start journalnode。

发现node2和node3节点上的JournalNode也连带被启动了。

提示：也可以通过命令hadoop-daemon.sh start journalnode启动单个节点的JournalNode，注意是hadoop-daemon.sh，不是hadoop-daemons.sh，此时不会连带启动其他节点的JournalNode。

Active Namenode启动（HDFS HA）

在node1节点执行hadoop-daemon.sh start namenode命令（单独操作一个节点），启动一个namenode作为Active Namenode：

按照规划，另一个namenode位于node2，所以需要在node2节点上进行namenode同步操作（实际上就是将namenode数据复制到node2节点）。

Standby Namenode启动（HDFS HA）

在node2节点上启动另外一个namenode，将成为Standby Namenode。

可能存在的问题

两个namenode状态不能自动切换，或者都处于standby状态。

请认真检查hdfs-site.xml和core-site.xml配置文件，可能存在配置项错误，比如某一项参数我们认为配置正确了，可能某个单词拼写错了，或者缺少 <name> 元素。
