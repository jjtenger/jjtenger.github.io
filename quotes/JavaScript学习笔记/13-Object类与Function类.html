create()方法

create方法，是除了对象字面量与new表达式之外的第三种官方的对象生成方法。它的第一个参数指定一个原型对象，第二个参数是一个关联数组，其键为属性名，值为属性描述符。属性描述符指的是属性对象及属性的属性（表一）中由属性的属性组成的关联数组。

如果将一个null作为原型对象传递给create方法，则会生成一个没有进行原型继承的对象。

Object类中对属性对象的操作

访问器属性：get、set

只要将get属性与set属性指定为相应的函数，就能够定义一个只能够通过访问器getter和setter来访问值的属性。

访问器与value属性是相互排斥的，也就是说，如果指定了value属性的值，访问器（同时包括get和set）就会失效；反之，如果指定了访问器（get或set中的某一个），value属性就会失效。

建议尽可能不使用不可变对象。

函数对象属性与内部变量的区别

函数的属性来自于函数对象本身，或者继承自Function.prototype对象的属性，比如：length、name属性和apply()和call()方法。而函数内部变量是其内部可执行代码的一部分，可以看作调用函数时被隐式生成的对象（Call对象）的属性。前者作为函数对象的属性，外部可以访问；后者作为局部变量，在其函数作用域内，外部无法访问。

注意：Function类本身也是Function类的对象实例。它们之间具有如下自我引用的关系。

这就意味着，对于一个函数，其原型继承于Function对象（Function类）的，可以读取访问Function.prototype对象的所有属性（或是对其进行方法调用）。

apply与call方法

Function对象中包含apply与call这两种方法，用来显式地指定接收方对象。即通过它们调用的函数的this引用，可以指向任意特定的对象。

apply与call方法中，this引用的是作为apply/call的第一个参数被传递的对象。而apply与call之间的不同之处在于两者对其他参数的传递方式。对于apply来说，剩余的参数将通过数组来传递，而call是直接按原样传递形参。
